# Geo Capture 0.1.13 Mobile Regression Checklist

## Core Commands

- [ ] `Quick insert place` opens and inserts a selected place.
- [ ] `Insert current location` inserts coordinates on mobile.
- [ ] `Capture nearby place` works with a valid Google Places API key.
- [ ] `Capture nearby place` falls back to raw current coordinates when nearby search fails.
- [ ] `Insert location from clipboard` still parses supported map links or coordinates.

## Image Location Flow

- [ ] The same image can be used for nearby place suggestions 3 times in a row.
- [ ] The first successful photo lookup can come from local EXIF.
- [ ] A repeated lookup still works when Geo Capture cache is the active source.
- [ ] When Google Places is unavailable, the flow still offers the raw photo GPS location.
- [ ] When no nearby places are returned, the flow still offers the raw photo GPS location.

## Diagnostics

- [ ] Diagnostics clearly show which source is active: `EXIF`, `R2 metadata`, or `Geo Capture cache`.
- [ ] Diagnostics provide a useful next-step hint when no source is available.

## Insert Behavior

- [ ] Selecting the same place twice in the same note triggers duplicate protection instead of inserting a second copy.
- [ ] `Below image` insertion still places the snippet directly under the image.

## Release Gate

- [ ] `npm run check` passes before release.
- [ ] Desktop smoke test is still clean after mobile fixes.
